# RT_PROEZD: Структурированные билеты для подготовки к экзамену в Obsidian.

## 🚀 Установка

Для корректного отображения и работы со всеми функциями базы знаний выполните следующие шаги:

1. **Клонирование репозитория:**
   ```bash
   git clone https://github.com/evanvd/RT_PROEZD.git
   ```
Или загрузите zip архив на этой страничке
![[Pasted image 20260524201337.png]]
2. **Открытие в Obsidian:**
   - Откройте Obsidian.
   - Выберите "Открыть папку как хранилище".
   ![[Pasted image 20260524201536.png|300]]
   - Укажите путь к склонированной папке `RT_PROEZD` или распакованному архиву.
## 📂 Структура проекта

- `Программа/` — Основная рабочая папка. Содержит готовые билеты, оформленные по стандарту.
- `Петрович md/` — Исходные тексты глав учебника в формате Markdown.(Удобно использовать для notebooklm)
- `Материалы/` — Центральный узел ссылок на все ресурсы.
- `RT_PROEZD.md` — Главный навигационный файл со списком всех тем и статусами их готовности.
- `.obsidian/` — Конфигурация рабочего пространства (темы, плагины, сниппеты).
## 🖋 Стандарты форматирования

Все билеты в папке `Программа/` следуют строгим правилам:

1.  **Callout-блоки:**
    - `[!info]` — Определения.
    - `[!error]` — Теоремы.
    - `[!note]` — Леммы.
2.  **Доказательства:** Всегда выносятся за пределы callout-блока и начинаются с `**Доказательство:**`.
3.  **Идентификаторы:** Каждое утверждение имеет уникальный якорь в конце блока (напр. `^th12d1`).
4.  **LaTeX:** Многострочные формулы объединяются в блоки `$$` для корректного рендеринга.
5.  **Ссылки:** Используются внутренние ссылки Obsidian `[[#^ID|Название]]` для быстрой навигации внутри файла.
